## 系统架构

### 后端
- **框架**：Express.js
- **数据库**：SQLite（便于部署）
- **端口**：3000（可环境变量配置）

### 前端
- **框架**：Vue 3
- **构建工具**：Vite
- **端口**：5173
- **API 调用**：Axios

### API 接口
见 `API.md`（如有）或代码注释

## 配置项

### 工作时间
- 通过店铺管理页面设置每个店铺的营业时间（上班、下班）
- 默认：09:00 ~ 18:00
- 用于判断迟到/早退

### 数据库
- SQLite 文件位置：`attendance.db`（项目根目录）
- 自动初始化表结构
- 支持数据迁移（动态添加字段）

## 使用步骤

### 1. 初始化店铺和员工
- 前往"店铺管理"，确认/修改店铺信息
- 设置每个店铺的营业时间（如果需要）
- 前往"员工管理"，添加员工或批量导入

### 2. 员工打卡
- 员工访问打卡页面
- 扫描自己的二维码
- 系统自动判断上班/下班

### 3. 查看统计
- 日报：查看当日出勤情况
- 月报：查看月度统计
- 工作时长：查看每个员工的工作时长

### 4. 数据导出
- 在工作时长页面点击"导出 Excel"
- 生成 CSV 文件（可用 Excel 打开）

## 部署注意事项

### 本地开发
```bash
# 后端
cd backend && npm start

# 前端（另一个终端）
cd frontend && npm run dev
```

### 云部署（Render/阿里云等）
- 后端：Node.js 环境，启动命令 `node server.js` 或 `npm start`
- 前端：静态网站，构建输出 `frontend/dist`
- 数据库：SQLite 文件持久化到磁盘

### 环保和长期使用
- SQLite 数据库自动保存到 `attendance.db`
- 不需要外部数据库，部署更简单
- 数据导出为 CSV，便于备份

## 已知限制

- 目前不支持多用户权限管理（任何人都可以修改数据）
- 二维码生成使用第三方免费 API（qrserver.com）
- 前端没有身份验证

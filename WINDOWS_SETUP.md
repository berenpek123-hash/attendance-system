# Windows 本地部署指南

## 准备工作

### 1. 安装 Node.js

1. 访问 [nodejs.org](https://nodejs.org)
2. 下载 **LTS 版本**（推荐 18.x 或 20.x）
3. 双击安装程序，点击"Next"逐步安装
4. **重要**：勾选"Add Node.js to PATH"
5. 完成后，重启电脑

### 2. 验证安装

打开命令提示符（Win+R，输入 `cmd`），运行：

```
node --version
npm --version
```

都显示版本号就说明安装成功了。

---

## 安装和启动系统

### 方法 1：双击启动脚本（最简单）

1. 找到 `windows-start.bat` 文件
2. 双击运行
3. 系统会：
   - 启动后端服务器
   - 启动前端服务器
   - 自动打开浏览器

### 方法 2：手动启动（需要两个命令窗口）

**第一个窗口（启动后端）：**
```
cd backend
node server.js
```

**第二个窗口（启动前端）：**
```
cd frontend
npm install
npm run dev
```

## 首次使用注意

### 首次启动时

第一次运行 `npm install` 可能需要 2-5 分钟，这是正常的。

系统提示：
```
✓ 服务器运行在 http://localhost:3000
✓ 数据库已初始化
```

说明后端已启动成功。

### 打开系统

在浏览器地址栏输入：

**管理员登陆页面：**
```
http://localhost:5173
```

**默认账号密码：**
- 用户名：`admin`
- 密码：`newpass123`（或你修改的密码）

**员工打卡页面：**
```
http://localhost:5173/checkin
```

## 日常使用

### 启动系统
1. 双击 `windows-start.bat`
2. 等待两个命令窗口打开
3. 浏览器自动打开登陆页面

### 关闭系统
直接关闭两个命令窗口即可。

### 数据保存
所有数据都保存在本地数据库文件（`attendance.db`），不会丢失。

## 常见问题

### Q：双击 .bat 文件后闪退
**A：** 通常是 Node.js 未安装或未添加到 PATH。
- 重新安装 Node.js，**勾选"Add Node.js to PATH"**
- 重启电脑
- 再试一次

### Q：显示"端口被占用"错误
**A：** 说明某个程序已经占用了端口。
- 关闭所有命令窗口
- 重启电脑
- 再试一次

### Q：启动后浏览器无法打开页面
**A：** 
- 检查命令窗口中是否显示"服务器运行在..."
- 如果显示了，等 5 秒再刷新浏览器
- 如果没显示，查看错误信息

### Q：忘记密码怎么办？
**A：** 
- 在登陆页面点击"忘记密码？"
- 输入用户名（admin）
- 系统生成临时密码
- 用临时密码登陆后修改密码

### Q：系统速度很慢
**A：** 这是正常的，因为是本地运行。如果实在慢，可以：
- 关闭其他程序释放内存
- 重启电脑
- 升级到 SSD（如果还是机械硬盘）

### Q：可以同时在多个电脑上运行吗？
**A：** 可以。每台电脑都需要：
- 安装 Node.js
- 复制项目文件夹
- 运行 `windows-start.bat`

## 备份和迁移

### 备份数据
数据都保存在 `attendance.db` 文件中，定期复制即可。

**备份方法：**
1. 找到 `attendance.db` 文件
2. 复制到 U 盘或云盘
3. 做好日期标记

### 恢复数据
1. 复制备份的 `attendance.db` 文件
2. 覆盖原来的文件
3. 重启系统

## 生产环境建议

如果需要长期稳定运行，建议：

1. **购买云服务器**（¥30-50/月）
   - 系统 7x24 运行
   - 多店铺可共享访问
   - 数据更安全

2. **使用内网穿透**（如 Frp）
   - 保留本地数据库
   - 外网可访问

3. **定期备份**
   - 每周备份数据库
   - 防止数据丢失
